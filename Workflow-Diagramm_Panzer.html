📋 Entwicklungs-Workflow

Code-Änderungen deployen
1
Claude ändert den Code — Du beschreibst was du willst, Claude bearbeitet die Dateien direkt in deinem Panzerspiel-Ordner.
2
Dateien auf dem PC aktualisiert — Die geänderten Quelldateien liegen lokal auf deinem Computer.
3
Doppelklick auf „Deploy zu Netlify.bat" — Das Skript führt automatisch git add · git commit · git push aus.
4
GitHub empfängt den Code — Der neue Code wird im Repository armor-ace gespeichert.
5
Netlify baut automatisch — Netlify erkennt den Push, baut das Spiel und stellt es live — dauert ca. 1–2 Minuten.
Spieler-Daten (geplant)
6
Spieler öffnen das Spiel im Browser — Das Spiel läuft direkt im Browser, auf jedem Gerät und Browser weltweit.
7
Supabase speichert Spielerdaten — Beim Anlegen eines Spielers, nach jedem Level und beim Upgrade werden Name, Level-Fortschritt und Score in der Online-Datenbank gespeichert — geräteübergreifend verfügbar.
